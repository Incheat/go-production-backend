# Documentation Guidelines

## Purpose

Ensure documentation is clear, consistent, and discoverable.

## Principles

* Use Markdown for simplicity and readability.
* Prefer short sections with clear headings.
* Keep examples close to explanations.

## Structure

* **Overview** — what the doc covers
* **Concepts** — definitions
* **Procedures** — step-by-step instructions
* **Examples** — practical usage

## Style Tips

* Use active voice.
* Keep sentences concise.
* Use code blocks for commands.
